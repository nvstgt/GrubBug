A01-2021: Broken Access Control

Broken access control is a vulnerability that occurs when users can gain unauthorized access to resources or actions beyond their intended permissions.

A02-2021: Cryptographic Failures

Insufficient or improper encryption leads to exposure of sensitive data or allows attackers to manipulate or steal data.

A03-2021: Injection

Malicious input, like SQL or command injection, tricks the application into executing unintended commands or accessing data without proper authorization.

A04-2021: Insecure Design

Applications lack secure design principles, leaving them vulnerable to exploitation due to inherent flaws in architecture or logic.

A05-2021: Security Misconfigurations

Improperly configured security settings, such as default credentials or overly permissive access controls, create vulnerabilities.

A06-2021: Vulnerable and Outdated Components

Using outdated libraries or software introduces known vulnerabilities into the application.

A07-2021: Identification and Authentication Failures

Weak authentication mechanisms or improperly implemented session management allow attackers to impersonate users.

A08-2021: Software and Data Integrity Failures

Trusting unverified software updates or external resources exposes the application to tampering and supply chain attacks.

A09-2021: Security Logging and Monitoring Failures

Inadequate logging or monitoring of activities prevents detection of breaches or malicious behavior in time to mitigate damage.

A10-2021: Server-Side Request Forgery (SSRF)

Applications fetch resources without proper validation, allowing attackers to force server requests to unintended locations, often bypassing firewalls.
